---
slug: user-mode
title: 用户态
description: 应用程序的运行模式，和内核态相对。
---

用户态是应用程序运行的非特权模式。在这种模式下：
1. 程序只能访问自己的虚拟地址空间
2. 不能直接访问硬件设备
3. 需要通过系统调用切换到内核态来获取特权服务
4. 大多数应用程序代码都在用户态下运行，保证了系统的安全性和稳定性

---
slug: kernel-mode
title: 内核态
description: 操作系统内核运行的特权模式。
---

操作系统内核运行的特权模式。在这种模式下：
1. CPU 可以执行所有特权指令
2. 可以直接访问所有物理内存和外设
3. 可以管理进程、线程、内存、文件系统等系统资源
4. 处理中断和异常
5. 提供系统调用服务

用户程序通过系统调用（System Call）从用户态切换到内核态。

---
slug: virtual-address-space
title: 虚拟地址空间
description: 操作系统为进程分配的独立内存空间。
---

操作系统为每个进程分配的逻辑内存空间。主要特点：
1. 每个进程都有独立的虚拟地址空间，相互隔离
2. 通过页表机制映射到物理内存
3. 典型的虚拟地址空间包含：
   - 代码段（存放程序代码）
   - 数据段（存放全局变量）
   - 堆（动态分配的内存）
   - 栈（局部变量和函数调用）
   - 共享库映射区
   - 内核空间

---
slug: page-table
title: 页表
description: 虚拟地址和物理地址的映射关系表。
---

页表是实现虚拟内存的核心机制：
1. 将虚拟地址空间分割成固定大小的页
2. 将物理内存分割成相同大小的页框
3. 页表记录虚拟页到物理页框的映射关系
4. 通常采用多级页表结构减少内存占用
5. 由 MMU（内存管理单元）硬件支持地址转换
6. 包含访问权限、是否存在、是否修改等标志位

---
slug: kernel-stack
title: 内核栈
description: 进程执行状态的记录区域。
---

每个进程都有自己的内核栈，用于：
1. 保存进程在内核态执行时的上下文
2. 存储系统调用的参数和返回值
3. 保存中断和异常处理的现场
4. 内核函数调用的栈帧
5. 切换进程时保存和恢复处理器状态

---
slug: program-counter
title: 程序计数器
description: 指向当前执行指令的地址指针。
---

程序计数器（PC）在 Java 中的特点：
1. 每个线程都有独立的 PC 寄存器
2. 如果执行 Java 方法，PC 记录正在执行的虚拟机字节码指令的地址
3. 如果执行 native 方法，PC 值为 undefined
4. 是实现线程切换和恢复的关键
5. 对于 Java 来说是唯一一个没有 OutOfMemoryError 的内存区域

---
slug: jvm-stack
title: 虚拟机栈
description: Java 线程执行的栈内存区域。
---

JVM 栈是线程私有的内存空间，包含：
1. 栈帧（Stack Frame）组成：
   - 局部变量表
   - 操作数栈
   - 动态链接
   - 方法返回地址
2. 特点：
   - 生命周期与线程相同
   - 可能抛出 StackOverflowError
   - 可以动态扩展（依赖于具体 JVM 实现）
3. 常见问题：
   - 递归调用过深导致栈溢出
   - 大量线程导致内存不足

---
slug: mutex-lock
title: 互斥锁
description: 线程同步的基本机制。
---

Java 中的互斥锁实现：
1. 基于 Monitor 机制：
   - 每个对象都有一个 Monitor
   - synchronized 关键字自动关联 Monitor
   - Monitor 由操作系统实现，由 JVM 映射到操作系统
2. 实现原理：
   - monitorenter：获取锁
   - monitorexit：释放锁
   - 锁计数器
3. 锁优化：
   - 偏向锁：减少无竞争情况下的开销
   - 轻量级锁：适用于线程交替执行同步块
   - 重量级锁：有线程竞争时升级
4. 特点：
   - 可重入性
   - 自动释放
   - 非公平性（可以配置为公平锁）

---
slug: jvm-reference
title: JVM 引用回收机制
description: Java 虚拟机的对象引用类型。
---

Java 的引用类型分为四种：

1. 强引用（Strong Reference）：
   - 最常见的引用类型
   - 只要强引用存在，对象就不会被回收
   - 即使发生 OOM 也不会回收
   - 例如：`Object obj = new Object()`

2. 软引用（Soft Reference）：
   - 用于内存敏感的缓存
   - 内存不足时会被回收
   - 通过 `SoftReference` 类实现
   - 常用于图片缓存、网页缓存等

3. 弱引用（Weak Reference）：
   - 比软引用更弱
   - 下一次 GC 时必定回收
   - 通过 `WeakReference` 类实现
   - 常用于 WeakHashMap 等容器

4. 虚引用（Phantom Reference）：
   - 最弱的引用关系
   - 随时可能被回收
   - 必须和引用队列（ReferenceQueue）一起使用
   - 用于跟踪对象被回收的状态
   - 通过 `PhantomReference` 类实现
